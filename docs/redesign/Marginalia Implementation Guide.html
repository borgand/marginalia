Marginalia — Implementation Guide

Building the redesigned UI on the IntelliJ Platform (Swing + UI DSL v2).

Companion to “Marginalia Redesign.html” + “marginalia-tokens.json” · New UI · light + dark via JBColor

Design intent. Keep everything inside what the Platform natively supports — no web views, no custom window chrome. The refresh is achieved entirely through standard components, JBColor theming, a custom list cell renderer, gutter highlighters, and a redesigned DialogWrapper. Every color is a role-named token that dereferences to a Platform theme key — so the same mock holds in both light and dark with zero hand-tuning.

00Principles

  • Name by role, never by appearance. The token is status.pending, not amber or color-3. The hue changes per theme; the role doesn’t. One id = one IDE lookup = both modes — never a pending.light and a pending.dark.
  • Zero raw hex on layers. Every fill, stroke and text color references a token. A literal hex on a layer is a handoff bug. Translucency is token @ NN% (the queued tint is status.pending @ 12%), never a new token.
  • Surface by what it sits on, not how it looks. Anything floating over the editor (add-comment toolbar, gutter highlight, applied-hunk flash) uses surface.editor; the tool window and its rows use surface.toolWindow. They look almost identical but resolve to different theme sources.
  • Margin-ink semantics. status.pending = queued / pending note. accent = delivered to agent. status.resolved = resolved. The same three roles appear in the list, the gutter, and the icon badge.
  • Don’t fight the editor. Back-references are drawn with the MarkupModel / gutter APIs the editor already exposes — not an overlay. Editor syntax colors come from the active EditorColorsScheme, not from the UI tokens.

★
Put the dereference in the token’s description so it ships inside the design file (e.g. description = JBUI.CurrentTheme.Link.Foreground.ENABLED). States are suffixes only when a real theme key exists; otherwise express as an alpha/blend of the base (a pressed row is selection.bg @ 80%).

Type

  • UI font — the IDE default (JBFont.regular(), which is Inter in the New UI). Don’t set a family; inherit it.
  • Code / anchors — the editor font, via EditorColorsManager.getInstance().getGlobalScheme().getEditorFontName() (JetBrains Mono by default).
  • Sizes — body JBFont.regular(), meta JBFont.small(), the L13 chip uses JBFont.smallOrNewUiDefault().

02The sidecar (tool window)

Container

Replace the current free-form panel with a SimpleToolWindowPanel(true, true). It gives you a real toolbar slot on top and a content area below.

class MarginaliaToolWindowFactory : ToolWindowFactory, DumbAware {
  override fun createToolWindowContent(project: Project, tw: ToolWindow) {
    val panel = MarginaliaPanel(project)            // SimpleToolWindowPanel
    val content = ContentFactory.getInstance()
        .createContent(panel, null, false)
    tw.contentManager.addContent(content)
  }
}

Header & toolbar

  • Put the title + connection chip + search + overflow in the tool-window header actions via ToolWindowEx.setTitleActions(...) and setAdditionalGearActions(...) — that keeps the chrome native. The amber brand mark is a 16×16 Icon set with toolWindow.setIcon(...).
  • The Auto-queue toggle + Submit review live in the setToolbar() row. Build it with an ActionToolbar from a DefaultActionGroup: a ToggleAction for Auto-queue, then the Submit action.
  • Submit review as a filled button (accent.button) is the one deliberate non-default accent. Use a JButton with putClientProperty("JButton.buttonType", "default") + ClientProperty.put(it, DarculaButtonUI…), or simpler, keep it a standard toolbar action with a tinted icon if you want to stay maximally conservative.
  • Move Restart server / Test connectivity into the gear/overflow menu — they don’t need to be primary.
★
Connection chip. The green “Connected” pill replaces the console as the at-a-glance signal. Render it as a small JBLabel with a round Icon dot + rounded background (RoundedLineBorder + isOpaque), placed as a title action. Drive its color/text from the MCP connection state.

Review-session progress ribbon

A thin custom component under the toolbar. Subclass JComponent and paint three rounded segments (resolved / delivered / queued) proportional to counts, then a legend row of JBLabels. Height JBUI.scale(5) for the bar.

The comment list — grouped by file

This is the core change. Two viable approaches:

  • Recommended — flattened JBList with a custom renderer. Build one ListModel mixing two row kinds: FileHeaderRow and CommentRow. A single ListCellRenderer switches layout on the kind. Gives you full control of the card styling and is far lighter than a tree.
  • Alternative — Tree / com.intellij.ui.dsl collapsible rows. Use if you need expand/collapse per file out of the box (CollapsibleRow in UI DSL v2).

Each comment card renderer paints, top to bottom:

  • A 2.5px status accent bar on the left edge (the row’s border = JBUI.Borders.customLine(statusColor, 0,2,0,0) won’t round — paint it in paintComponent for the rounded look).
  • Row 1: status pill (rounded JBLabel, soft bg) + line chip L13 + spacer + hover-only jump / more action icons (show via list.getMousePosition hover state).
  • Row 2: anchor snippet in the editor font, muted, single-line with … ellipsis, a 2px left rule.
  • Row 3: comment text, UI font, wraps to ~2 lines.
  • Row 4: avatar (initials in a circle) + You · 2m ago.

The file header row: disclosure chevron + markdown FileTypeManager icon + filename (bold) + count badge + a status.pending “N to send” indicator when queued>0.

Jump-to-line (back-reference)

// on click / Enter / the row's jump icon
val file = LocalFileSystem.getInstance().findFileByPath(comment.path)
OpenFileDescriptor(project, file, comment.line, 0)
    .navigate(true)        // focuses editor + scrolls to the anchored line

Footer status panel

A one-line bar: green dot + Agent connected · last call 56s ago + an MCP :4747 chip + a disclosure chevron. The chevron toggles a collapsed JBScrollPane holding the raw connectivity log (the current console) — kept, but tucked away by default.

03Comment box — modal

Rebuild “Add Marginalia Comment” as a styled DialogWrapper.

  • Context strip at the top: file icon + README.md + line 13, then the anchored line rendered in the editor font inside a status.pending-ruled quote box. Pull the snippet from the Editor/Document at the caret line — this is what makes the dialog feel anchored.
  • Input: a JBTextArea in a JBScrollPane, or an EditorTextField with markdown language for live highlighting. Give it the focus ring by relying on the default focus border (don’t custom-paint).
  • Footer: a status.pending “Queues to the agent” hint on the left; createActions() returns [Cancel, OK] where OK is the default and relabelled “Add comment”. Bind ⌘↵ to OK via setOKButtonMnemonic / a registered shortcut.
★
Keep the native frame. Don’t draw your own titlebar — DialogWrapper supplies the OS window chrome (the traffic lights in the mock are macOS’s own). Set title = "Add comment" and isResizable = true.

Comment box — inline popup optional / nicer

For an in-flow capture that never leaves the editor, present the same content as a balloon anchored to the caret line via JBPopupFactory:

val popup = JBPopupFactory.getInstance()
  .createComponentPopupBuilder(commentForm, commentForm.textArea)
  .setRequestFocus(true)
  .setResizable(true)
  .setCancelOnClickOutside(false)
  .createPopup()
popup.showInBestPositionFor(editor)   // points at the caret line

The balloon shows: pen icon + “Comment” + README.md : 25 chip + a status.pending “Queued” tag, the input, then Esc to cancel / Add ⌘↵. Offer both surfaces; let a setting pick the default.

04In-editor back-referencing

Three native layers make commented lines legible inside the document:

  • Gutter pen icon — for each comment, add a RangeHighlighter on its line and attach a GutterIconRenderer that paints the status.pending pen and exposes a click action (open the comment / focus the sidecar row).
  • Line tint — same highlighter carries TextAttributes with a faint status.pending @ 12% backgroundColor and a status.pending errorStripeColor so the line also shows on the scrollbar map.
  • Tab affordance — a literal dot on a tab isn’t a native primitive; instead tint the tab via EditorTabColorProvider (a barely-there status.pending wash) so files with open comments stand out. This is fully supported and theme-safe.
val markup = editor.markupModel
val hl = markup.addLineHighlighter(line, HighlighterLayer.SELECTION - 1, attrs)
hl.gutterIconRenderer = object : GutterIconRenderer() {
  override fun getIcon() = MarginaliaIcons.PenAmber
  override fun getClickAction() = openCommentAction(comment)
  override fun getTooltipText() = comment.text
  override fun equals(o: Any?) = …   override fun hashCode() = …
}

Drive these highlighters off the comment model and the open FileEditorManager editors; refresh on comment add/resolve.

05Tool-window stripe icon & badge

  • Base icon — the status.pending margin-rule mark as a 16×16 themed Icon (@2x + dark variants via the standard naming, or an SvgIcon).
  • Needs-review badge — overlay the queued count with com.intellij.ui.BadgeIconSupplier (or a LayeredIcon + small TextIcon), then toolWindow.setIcon(badged) whenever the queued count changes. status.pending badge = pending notes.
  • Optionally flash the stripe via ToolWindowManager…getToolWindow("Marginalia").activate-free attention by setting the icon — avoid stealing focus.

08Build checklist

  1. Register tokens via *.themeMetadata.json for light/dark and ship marginalia-tokens.json; bind each api into MarginaliaColors.
  2. Swap the panel to SimpleToolWindowPanel; move title/connection/overflow into tool-window header actions.
  3. Build the toolbar (Auto-queue toggle, Submit review) and the progress ribbon component.
  4. Replace the JTable with a flattened JBList + grouped renderer (file headers + comment cards).
  5. Wire jump-to-line via OpenFileDescriptor; add hover jump/more actions.
  6. Add the footer status panel with collapsible raw log.
  7. Redesign the DialogWrapper with the anchored-snippet context strip; add the optional inline JBPopup.
  8. Add gutter highlighters + GutterIconRenderer + EditorTabColorProvider for back-references.
  9. Badge the stripe icon from the queued count.
  10. Add the subtle state-transition animations last.
